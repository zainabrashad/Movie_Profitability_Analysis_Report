Assessment 3 – Database Schema, Population & SQL Queries¶

Short Summary This project focuses on the design and implementation of a normalized relational database for movie data and performing analytical queries using SQL and Python. The database was structured into six related tables to ensure data consistency, reduce redundancy, and improve query efficiency.

The dataset was imported from an Excel file and cleaned before being transformed into a normalized format. Categorical attributes such as genres and production companies were separated into individual lookup tables, while many-to-many relationships were managed through junction tables. The processed data was then stored in an SQLite database.

A series of SQL queries were developed using joins, aggregation, filtering, and sorting functions to extract meaningful insights from the dataset. Additionally, database views were created to simplify complex queries and support reusable analysis.

The database schema was designed and visually represented using dbdiagram.io, where all tables and their relationships were defined in an Entity-Relationship (ER) diagram. The diagram includes seven normalized tables: movies, ratings, genres, production_companies, movie_genres, wikipedia info and movie_companies. Primary keys and foreign key relationships were clearly mapped to show how data flows between tables. This visual representation helped in understanding the structure of the database and ensured proper normalization before implementation in SQLite.

Finally, data visualization techniques were applied using Python to present key insights, including top profitable movies and top revenue-generating production companies, in a clear graphical format.

import pandas as pd
import sqlite3

# 1. LOAD DATA
df = pd.read_excel('movies_final_output.xlsx')
df.columns = df.columns.str.strip()
df = df.reset_index(drop=True)
df['movie_id'] = df.index

# 2. MOVIES TABLE
movies_df = df[['movie_id', 'title', 'budget', 'revenue', 'profit', 'year']]

# 3. RATINGS TABLE
ratings_df = df[['movie_id', 'api_rating', 'vote_count', 'api_popularity']]

# 4. WIKIMEDIA TABLE
wikimedia_df = df[['movie_id', 'wiki_page_length']].copy()
wikimedia_df['wiki_id'] = wikimedia_df.index
wikimedia_df = wikimedia_df[['wiki_id', 'movie_id', 'wiki_page_length']]

# 5. GENRES
genres = df[['movie_id', 'genres']].dropna()
movie_genres_df = genres.assign(genre=genres['genres'].str.split(',')).explode('genre')
movie_genres_df['genre'] = movie_genres_df['genre'].str.strip()
genres_df = pd.DataFrame(movie_genres_df['genre'].unique(), columns=['genre_name'])
genres_df['genre_id'] = genres_df.index
movie_genres_df = movie_genres_df.merge(genres_df, left_on='genre', right_on='genre_name')[['movie_id', 'genre_id']]

# 6. PRODUCTION COMPANIES
companies = df[['movie_id', 'production_companies']].dropna()
movie_companies_df = companies.assign(company=companies['production_companies'].str.split(',')).explode('company')
movie_companies_df['company'] = movie_companies_df['company'].str.strip()
companies_df = pd.DataFrame(movie_companies_df['company'].unique(), columns=['company_name'])
companies_df['company_id'] = companies_df.index
movie_companies_df = movie_companies_df.merge(companies_df, left_on='company', right_on='company_name')[['movie_id', 'company_id']]

# 7. CREATE SQLITE DATABASE WITH FOREIGN KEYS AND INDEXES
conn = sqlite3.connect('movies_normalized.db')
cursor = conn.cursor()
cursor.execute("PRAGMA foreign_keys = ON;")

cursor.executescript('''
DROP TABLE IF EXISTS movie_genres;
DROP TABLE IF EXISTS movie_companies;
DROP TABLE IF EXISTS genres;
DROP TABLE IF EXISTS production_companies;
DROP TABLE IF EXISTS ratings;
DROP TABLE IF EXISTS wikimedia_info;
DROP TABLE IF EXISTS movies;

CREATE TABLE movies (
    movie_id INTEGER PRIMARY KEY,
    title TEXT NOT NULL,
    budget INTEGER,
    revenue INTEGER,
    profit INTEGER,
    year INTEGER
);

CREATE TABLE ratings (
    movie_id INTEGER,
    api_rating REAL,
    vote_count INTEGER,
    api_popularity REAL,
    FOREIGN KEY (movie_id) REFERENCES movies(movie_id) ON DELETE CASCADE
);

CREATE TABLE wikimedia_info (
    wiki_id INTEGER PRIMARY KEY,
    movie_id INTEGER NOT NULL UNIQUE,
    wiki_page_length INTEGER,
    FOREIGN KEY (movie_id) REFERENCES movies(movie_id) ON DELETE CASCADE
);

CREATE TABLE genres (
    genre_id INTEGER PRIMARY KEY,
    genre_name TEXT UNIQUE NOT NULL
);

CREATE TABLE production_companies (
    company_id INTEGER PRIMARY KEY,
    company_name TEXT UNIQUE NOT NULL
);

CREATE TABLE movie_genres (
    movie_id INTEGER,
    genre_id INTEGER,
    PRIMARY KEY (movie_id, genre_id),
    FOREIGN KEY (movie_id) REFERENCES movies(movie_id) ON DELETE CASCADE,
    FOREIGN KEY (genre_id) REFERENCES genres(genre_id) ON DELETE CASCADE
);

CREATE TABLE movie_companies (
    movie_id INTEGER,
    company_id INTEGER,
    PRIMARY KEY (movie_id, company_id),
    FOREIGN KEY (movie_id) REFERENCES movies(movie_id) ON DELETE CASCADE,
    FOREIGN KEY (company_id) REFERENCES production_companies(company_id) ON DELETE CASCADE
);

-- Indexes for performance
CREATE INDEX idx_ratings_movie_id ON ratings(movie_id);
CREATE INDEX idx_wiki_movie_id ON wikimedia_info(movie_id);
CREATE INDEX idx_movie_genres_movie_id ON movie_genres(movie_id);
CREATE INDEX idx_movie_genres_genre_id ON movie_genres(genre_id);
CREATE INDEX idx_movie_companies_movie_id ON movie_companies(movie_id);
CREATE INDEX idx_movie_companies_company_id ON movie_companies(company_id);
''')

# 8. INSERT DATA
movies_df.to_sql('movies', conn, if_exists='append', index=False)
ratings_df.to_sql('ratings', conn, if_exists='append', index=False)
wikimedia_df.to_sql('wikimedia_info', conn, if_exists='append', index=False)
genres_df.to_sql('genres', conn, if_exists='append', index=False)
companies_df.to_sql('production_companies', conn, if_exists='append', index=False)
movie_genres_df.to_sql('movie_genres', conn, if_exists='append', index=False)
movie_companies_df.to_sql('movie_companies', conn, if_exists='append', index=False)

conn.commit()
conn.close()

print("Database created successfully with foreign keys and indexes.")

Database created successfully with foreign keys and indexes.

This code loads a movie dataset from an Excel file and cleans the column names. It then splits the data into multiple normalized tables for movies, ratings, genres, and production companies. It also creates junction tables to handle many-to-many relationships between movies and genres, and movies and companies. After that, it creates a SQLite database, defines all required tables with relationships and indexes, and inserts the processed data into them. Finally, it saves and closes the database, resulting in a fully normalized and optimized movie database ready for analysis.

Conclusion & Key Insights¶

In this project, a normalized SQLite database was successfully designed and implemented from a raw movie dataset. The data was cleaned, transformed, and loaded into a structured relational schema, followed by extensive SQL-based analysis to extract meaningful insights.

Work Completed Designed a 7-table normalized schema including movies, ratings, genres, production_companies, wiki pedia impact on profit,movie_genres, and movie_companies with proper primary keys, foreign keys, and indexing. Loaded and transformed data from an Excel file into SQLite using Python and Pandas. Developed multiple SQL queries using joins, aggregation, filtering, and sorting to address analytical questions. Created database views to simplify complex queries and generated visualizations using Python.

Key Insights

Profitability by Movie: Avatar is the most profitable movie with over $2.5 billion profit, followed by Titanic and Jurassic World. All top movies show exceptionally high profitability.

Profitability by Genre: Animation and Adventure genres generate the highest average profits, while History and Western genres are the least profitable.

Top Production Companies by Revenue: Warner Bros. is the leading company with the highest total revenue, followed by major studios such as Paramount, Universal, and Disney.

Critically Acclaimed but Unprofitable Movies: Some highly rated movies (rating > 7.0), such as The Iron Giant and Children of Men, performed poorly financially, showing that critical success does not always guarantee profit.

Year-wise Trends: Movie production increased significantly after the 1990s, and average popularity has generally risen over time, peaking in recent years.

Genre Popularity: Science Fiction and Adventure are the most popular genres, while Western and War genres show lower audience engagement.

Wikipedia Info vs Movie Performance Insight Movies with Medium and High Wikipedia page length generally have higher revenue and profit, showing a positive relationship between information level and movie performance. However, the pattern is not perfect, as some low-info movies also perform well.

Overall, this project demonstrates how structured databases and SQL analysis can transform raw data into valuable business insights for decision-making.
